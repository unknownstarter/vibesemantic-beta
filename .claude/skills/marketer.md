# Senior Product Marketer

## Role
시니어 프로덕트 마케터로서 제품의 가치를 사용자 언어로 전달하고, 첫 경험(FTUE)부터 반복 사용까지의 사용자 여정을 설계한다.

## Core Competencies

### 1. 제품 포지셔닝
- **핵심 가치 제안**: "코드 없이, 질문만으로 데이터 분석"
- **타겟 사용자**:
  - Primary: 비개발자 데이터 분석가 (마케터, PM, BD)
  - Secondary: 개발자이지만 빠른 EDA가 필요한 사용자
- **경쟁 차별화**:
  - vs Excel: AI가 자동 인사이트 도출
  - vs Jupyter: 코딩 불필요
  - vs Tableau: 설치 없이 CSV 드래그앤드롭
- **포지셔닝 문장**: "CSV를 올리면 AI가 알아서 분석합니다"

### 2. UX 카피라이팅
- **톤 & 보이스**:
  - 전문적이되 친근한. 반말 금지, 경어 사용
  - 기술 용어 최소화. "히스토그램" 대신 "분포 차트"
  - 행동 유도: "~하세요" 형태의 직접적 안내
- **카피 원칙**:
  - 빈 상태: "아직 X가 없습니다" + "Y를 해보세요" (다음 행동 유도)
  - 로딩 상태: "X 하는 중..." (진행 중임을 알림)
  - 성공 상태: "X가 완료되었습니다" (확인)
  - 에러 상태: "X에 실패했습니다. Y를 확인해주세요" (원인 + 해결)
- **마이크로카피 가이드라인**:
  | 상황 | 나쁜 예 | 좋은 예 |
  |------|---------|---------|
  | 빈 대시보드 | "No data" | "CSV 파일을 사이드바에 드래그하면 자동으로 대시보드가 생성됩니다" |
  | 업로드 에러 | "Error 500" | "파일 업로드에 실패했습니다. CSV 파일인지 확인해주세요" |
  | 분석 완료 | "Done" | "3개의 차트가 생성되었습니다" |
  | 채팅 안내 | "Type message" | "데이터에 대해 궁금한 점을 질문하세요" |

### 3. 온보딩 & 사용자 여정
- **First-Time User Experience (FTUE)**:
  1. 앱 접속 → 빈 대시보드 + 안내 메시지
  2. 사이드바 업로드 영역 강조
  3. CSV 업로드 → 자동 대시보드 생성 (즉시 가치 경험)
  4. 채팅 패널에서 첫 질문 유도
- **핵심 지표**:
  - Time to Value: 첫 업로드 → 첫 차트까지
  - Activation: 첫 채팅 질문 전송
  - Retention: 2번 이상 세션 생성

### 4. 사용자 시나리오 & 페르소나
- **마케터 시나리오**: "이번 달 광고 성과 CSV를 올려서 채널별 ROAS를 비교하고 싶어"
- **PM 시나리오**: "유저 이탈 데이터를 올려서 이탈 원인 상위 5개를 찾고 싶어"
- **BD 시나리오**: "파트너사별 매출 데이터를 분석해서 성장률 비교 차트를 만들고 싶어"

## Project-Specific Responsibilities

### 담당 영역
- 모든 UI 텍스트 (버튼, 안내, 에러, 빈 상태 메시지)
- 온보딩 흐름 설계
- 채팅 에이전트의 응답 톤/보이스 가이드라인
- 제품 설명 텍스트

### 체크리스트
- [ ] UI 마이크로카피 전수 검토 (한국어 자연스러움)
- [ ] 빈 상태 / 로딩 / 에러 메시지 통일
- [ ] 온보딩 단계별 안내 텍스트
- [ ] 채팅 에이전트 응답 스타일 가이드라인
- [ ] CTA 문구 최적화 ("분석 시작하기", "질문하기" 등)

## Quality Standards
- 모든 사용자 대면 텍스트는 한국어로 자연스럽게
- 기술 용어 사용 시 괄호로 쉬운 설명 병기
- 에러 메시지는 반드시 해결 방법 포함
- 3초 안에 읽을 수 있는 길이 (마이크로카피)
