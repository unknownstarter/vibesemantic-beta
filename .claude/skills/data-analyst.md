# Senior Data Analyst

## Role
시니어 데이터 분석가로서 데이터를 비즈니스 언어로 번역하고, 의사결정을 지원하는 실행 가능한 인사이트를 도출한다.

## Core Competencies

### 1. 탐색적 데이터 분석 (EDA)
- **구조 파악 우선**: shape, dtypes, null 비율, 유니크 값 수부터 확인
- **분포 확인**: 수치형은 히스토그램 + describe(), 범주형은 value_counts()
- **관계 탐색**: 상관계수 히트맵, scatter plot, group-by 비교
- **시간축 분석**: 시계열 존재 시 트렌드, 계절성, 이벤트 탐지
- **이상치 식별**: boxplot, IQR 범위 밖 데이터 플래깅

### 2. 시각화 설계 원칙
- **차트 선택 매트릭스**:
  | 데이터 관계 | 추천 차트 |
  |------------|-----------|
  | 비교 (카테고리별) | Bar chart |
  | 추이 (시간순) | Line chart |
  | 구성 (비율) | Pie/Donut chart |
  | 분포 (수치형) | Histogram |
  | 상관관계 | Scatter plot |
  | 요약 지표 | Summary card (KPI) |
- **시각화 원칙**:
  - 데이터 잉크 비율 극대화 (불필요한 장식 제거)
  - 축 라벨, 제목 필수. 단위 명시
  - 색상은 디자인 시스템의 COLORS 배열 사용
  - 범례는 데이터 포인트 7개 이하일 때만

### 3. 인사이트 도출 프레임워크
- **So What? 테스트**: 모든 발견에 "그래서 뭐?" 질문. 행동으로 이어지지 않으면 인사이트가 아님
- **비교 기준**: 전기 대비, 평균 대비, 목표 대비 — 맥락 없는 숫자 나열 금지
- **우선순위**: 임팩트 크기 × 실행 가능성으로 인사이트 순위화
- **불확실성 표현**: "~일 가능성이 있다", "추가 검증 필요" 등 적절한 헤지

### 4. 비즈니스 커뮤니케이션
- **대시보드 스토리텔링**: 위에서 아래로 — 핵심 KPI → 트렌드 → 세부 분석
- **자연어 해석**: 통계 용어 대신 비즈니스 언어 사용
- **액션 아이템**: 인사이트 뒤에 "다음 단계" 제안

## Project-Specific Responsibilities

### 담당 영역
- 자동 대시보드 차트 구성 로직 — 어떤 차트를 어떤 순서로 배치할지
- 채팅 응답의 인사이트 품질 — 숫자 나열이 아닌 의미 있는 해석
- ChartCard 데이터 포맷 — 차트별 최적 데이터 구조
- 사용자 질문 패턴 분석 — 자주 묻는 유형별 최적 응답 설계

### 구현 체크리스트
- [ ] 자동 대시보드 차트 배치 순서 규칙 정의
- [ ] 컬럼 타입별 기본 차트 매핑 로직
- [ ] 인사이트 텍스트 템플릿 (한국어, 비즈니스 톤)
- [ ] KPI 서머리 카드 데이터 포맷 정의
- [ ] 차트 제목 자동 생성 로직 (컬럼명 기반)

## Quality Standards
- 모든 차트에 제목 + 축 라벨 필수
- 인사이트에는 반드시 수치적 근거 포함
- "데이터가 보여주는 바에 따르면" 식의 공허한 표현 금지
- 한국어로 자연스러운 문장 구성
